Purdue Capstone — Azure Cloud Architecture

IT473 E-Commerce Platform for Food Distribution

Compute Layer

  • • Azure App Service (Linux) — B1 tier
  • • Django 5.1 + Gunicorn in Docker
  • • Auto-scaling + health checks
  • • Built-in HTTPS / custom domains
  • • Deployment slots for staging

Data Layer

  • • Azure SQL Database (S0 tier)
  • • Transparent Data Encryption (TDE)
  • • Firewall rules + IP whitelisting
  • • Auto-backups (7-day retention)
  • • Local fallback: PostgreSQL 16

Storage & Assets

  • • Azure Blob Storage (Unit 5+)
  • • Product images + static files
  • • CDN-ready for global delivery
  • • Azure Container Registry (ACR)
  • • Private Docker image storage

CI/CD Pipeline

  • • GitHub Actions workflow
  • • Auto-build on push to main
  • • Push image to ACR
  • • Deploy to App Service
  • • Rollback via slot swap

Security (Unit 6+)

  • • Azure Key Vault for secrets
  • • Managed Identity (no passwords in code)
  • • SQL Firewall + Private Endpoints
  • • TLS 1.2 minimum
  • • .env files never committed

Monitoring (Unit 7+)

  • • Azure Monitor + Log Analytics
  • • Application Insights for Django
  • • Custom alert rules
  • • Performance metrics dashboard
  • • Health check endpoint: /health/
